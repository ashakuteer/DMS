GO DONATIONS FORM IMPROVEMENTS (NON-CASH PAYMENT DISABLE + OPTIONAL QTY/UNIT)

Goal:
Improve the "Record Donation" modal on Donations page so it correctly handles non-cash (in-kind) donations.

Changes required:

1) Donation Type vs Payment Mode rules
- If Donation Type is "Cash" (or any money donation type):
  - Payment Mode stays enabled and required.
  - Allowed modes: Cash, UPI, GPay, PhonePe, Cheque, Online, Bank Transfer (existing list is fine).
- If Donation Type is anything else (Groceries, Rice Bags, Stationery, Medicines, Sports Kits, etc):
  - Payment Mode must be DISABLED (greyed out) and NOT required.
  - Automatically set Payment Mode value to "N/A" (or "IN_KIND") in UI and in payload.
  - Backend should accept this without validation error.
  - In Donation History, show Mode as "IN-KIND" (or blank) for non-cash donations.

Implementation details:
- Add a constant / enum value for "IN_KIND" (preferred) or "N/A" in the paymentMode options in both FE and BE.
- FE: watch donationType field; when it changes:
  - if donationType !== "Cash": set paymentMode="IN_KIND" and disable the field
  - if donationType === "Cash": enable the field and set default paymentMode="Cash" if previously IN_KIND.
- BE validation:
  - if donationType === "Cash": require paymentMode != IN_KIND
  - if donationType !== "Cash": allow paymentMode to be IN_KIND and do not require other modes

2) Quantity + Unit should be OPTIONAL
- In Record Donation modal:
  - Remove required validation for Quantity and Unit.
  - Allow empty/null values.
  - If user enters one, they can enter the other; but both are optional.
- Backend DB + API:
  - Ensure quantity and unit columns/fields allow null.
  - Prisma schema: make quantity and unit optional (nullable).
  - Update create donation DTO/validation accordingly.

3) Estimated Value (₹) remains OPTIONAL
- Keep Estimated Value optional (already).
- Rename label to "Approximate Value (₹) - Optional" in UI for clarity.
- Backend: accept null.

4) UX polish
- When Payment Mode is disabled (non-cash types), show helper text:
  “In-kind donation (no payment mode required)”
- Keep existing Designated Home dropdown.

After changes:
- Test creating:
  a) Cash donation with Payment Mode required.
  b) Rice Bags donation with Payment Mode disabled and saved as IN_KIND.
  c) Stationery donation with empty quantity/unit, should save successfully.

Do NOT change any other pages unless needed for compilation/build.
