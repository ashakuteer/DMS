You are working in the NGO DMS monorepo (Next.js in /apps/web, NestJS in /apps/api, Supabase Postgres). Implement “Sponsor Birthday Wish from Beneficiary Photo” end-to-end.

GOAL
On a DONOR/Sponsor birthday, automatically prepare a personalized birthday wish that includes:
- Sponsor name
- The sponsored beneficiary name(s)
- One beneficiary photo (or fallback to home/group photo)
- A short message template (editable)
- Actions: (1) Copy WhatsApp message (manual sending), (2) Queue Email (auto email via existing SMTP queue)

IMPORTANT RULES
- WhatsApp remains manual: generate text + a “copy” button only.
- Email can be automated using the existing EmailJobs queue/sender.
- Never use sensitive documents. Only allow photos marked non-sensitive and share_with_donor=true.
- If beneficiary privacy_protected=true, do NOT include their name/photo in donor birthday messages (use “our children at <Home>” with a generic group photo if available).

DATA MODEL CHANGES
1) Add fields to donors table (if not already):
- dob_day int nullable
- dob_month int nullable
(keep any existing special days structure but ensure donor birthday is queryable)

2) Add/extend documents metadata (if already exists) to support:
- doc_type = BENEFICIARY_PHOTO, HOME_PHOTO (group), ORG_EVENT_PHOTOS
- share_with_donor boolean default false
- is_sensitive boolean default false
Ensure photos used for donor wishes must have: is_sensitive=false AND share_with_donor=true.

3) Add table: message_templates
- id uuid
- key text unique (e.g., DONOR_BIRTHDAY_WISH)
- channel enum EMAIL, WHATSAPP
- subject text nullable (for email)
- body text (supports variables)
- updated_at, updated_by

Default templates:
WHATSAPP:
"Dear {{donor_name}},\n\nHappy Birthday! 🎉🎂\nOn this special day, {{org_name}} thanks you for your support.\n{{beneficiary_line}}\nWishing you a wonderful year ahead.\n\nWith gratitude,\n{{org_name}}"
EMAIL subject:
"Happy Birthday, {{donor_name}}! – {{org_name}}"
EMAIL body similar to WhatsApp but formatted with paragraphs and includes image.

VARIABLES
- {{donor_name}}
- {{org_name}} (Asha Kuteer Foundation)
- {{beneficiary_line}}:
  - If sponsor has 1 sponsored beneficiary visible: "Your sponsored beneficiary {{beneficiary_name}} sends warm wishes."
  - If multiple: "Your sponsored beneficiaries {{beneficiary_names}} send warm wishes."
  - If none or privacy-protected: "Our children and elders send warm wishes."

BACKEND
1) Create query: get donors with birthday in next N days (default 7) and today.
2) Create service: BirthdayWishService
- For a donor, fetch active sponsorships -> beneficiaries
- Filter out privacy_protected beneficiaries from naming/photo usage
- Choose photo:
   a) first eligible beneficiary photo: doc_type=BENEFICIARY_PHOTO AND share_with_donor=true AND is_sensitive=false
   b) else home/group photo: doc_type=HOME_PHOTO share_with_donor=true
   c) else no image (text only)
- Render templates with variables
- Return: whatsappText, emailSubject, emailHtml, imageDocumentId(optional)

3) API endpoints:
- GET /api/birthdays/donors?range=today|next7
  returns donors with computed wish preview + flags (hasEmail, hasWhatsApp)
- POST /api/birthdays/donors/:donorId/queue-email
  queues email using existing EmailJobsService; attaches image if available OR embeds signed image URL (preferred)
- POST /api/birthdays/donors/:donorId/mark-sent
  records an audit log row (below)

4) Add audit table: outbound_messages_log
- id uuid
- type enum DONOR_BIRTHDAY
- channel enum EMAIL, WHATSAPP
- donor_id uuid
- beneficiary_ids uuid[] nullable
- status enum QUEUED, SENT, COPIED
- created_at, created_by

FRONTEND (Daily Actions + Donor Profile)
1) Daily Actions page:
- Add section “Sponsor Birthdays (Next 7 Days)”
- Table columns: Donor, Date, Sponsored Beneficiary(ies), Actions
Actions:
  - “Copy WhatsApp Wish” button -> copies generated whatsappText and logs COPIED
  - “Queue Email Wish” button -> calls queue-email, toast “Email queued”
  - “Open Donor Profile” link
If donor has no email, disable Queue Email and show tooltip “No email on donor”.

2) Donor profile:
- Add tab “Sponsorships” (already) and add a mini panel “Birthday Wish Preview”
- Buttons same as above.

3) Templates page:
- Add UI to edit templates for DONOR_BIRTHDAY_WISH (Email + WhatsApp) with live variable hints.
- Save to DB via API.

EMAIL IMAGE HANDLING
- Do NOT expose private URLs.
- Use backend to generate a signed URL for the selected image and embed it in email HTML as <img src="SIGNED_URL"> with short expiry.
- If email client blocks images, message must still read well without image.

FINISHING
- Ensure this feature uses existing SMTP settings and email queue.
- Add seeded data: at least one donor with birthday today and one donor with birthday within 7 days; each with sponsorship to one beneficiary with an eligible shared photo.

After implementation, print:
1) Where in UI to test (exact pages)
2) Which template keys were created
3) Any required setup for photos to be shareable (share_with_donor toggle)
