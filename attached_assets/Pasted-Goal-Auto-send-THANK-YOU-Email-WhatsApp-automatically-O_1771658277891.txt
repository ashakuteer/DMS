Goal: Auto-send THANK YOU Email + WhatsApp automatically ONLY on first donation creation. Keep manual resend buttons as-is.

Current state:
- Donation Details modal already has manual resend buttons:
  "Send Thank You (Email)" and "Send Thank You (WhatsApp)" + "Download Receipt". DO NOT change these.
- There is also a "Donation Saved Successfully" popup after saving donation that currently shows buttons like "Send Thank-You Email" and "Send WhatsApp". We do NOT want these buttons for first-time sending.

What to implement (minimum changes):

A) Backend (NestJS) - auto send on donation creation
1. In apps/api/src/donations/donations.service.ts inside create() AFTER donation is successfully created in DB:
   - Trigger existing Email send logic (same logic used by the manual "Send Thank You (Email)" action) automatically.
   - Trigger existing Twilio WhatsApp template send (Content API contentSid) automatically (same logic used by the manual "Send Thank You (WhatsApp)" action) automatically.
2. If Email/WhatsApp fails, donation should still remain saved. Log errors; do not throw error back to client.
3. Add a simple "sent-once" guard so auto send runs only once per donation:
   - Use CommunicationMessage table if available (recommended): before auto-send, check if a CommunicationMessage exists for this donationId + channel EMAIL/WHATSAPP + type THANK_YOU. If exists, skip auto-send.
   - Otherwise add fields on Donation like emailThankYouSentAt and whatsappThankYouSentAt (but prefer CommunicationMessage since it already exists).
4. Do not modify manual resend endpoints/controllers except if you need to re-use them internally (call the same service methods).

B) Frontend - remove post-save buttons
1. In the "Donation Saved Successfully" modal (the popup shown immediately after saving donation):
   - Remove/hide the buttons: "Send Thank-You Email" and "Send WhatsApp" and "Remind Later" if present.
   - Keep only a success message + Close/Skip.
2. Manual resend remains available ONLY in Donation Details modal (already working).

Acceptance criteria:
- Save donation -> automatically sends email + WhatsApp in background.
- The post-save modal shows no send buttons.
- Donation Details modal manual resend buttons still work exactly as before.
