Goal: Add a new “Broadcasting” feature in the Dashboard to send WhatsApp or Email messages to a targeted set of donors using filters.

1) UI / Navigation
- Add a new sidebar/menu item and page: /dashboard/broadcasting
- Button/label: “Broadcasting”

2) Broadcasting Page – Required Features
A) Channel selection
- Choose channel: WhatsApp / Email (radio or dropdown)
- Use existing integrations:
  - WhatsApp: Twilio Content API templates (contentSid)
  - Email: existing email system (SMTP/email service already in project)

B) Template selection + variables
- Allow choosing one approved template (WhatsApp templates + email templates)
- Support basic variables like:
  - donorName
  - organizationName
  - donationAmount (optional)
  - date (optional)
- For WhatsApp, use Twilio Content API (contentSid) + contentVariables.

C) Audience selection (powerful filters)
- Show donor filter builder that can filter based on ANY donor profile field we already have.
- Must include common quick filters:
  - Gender (Male/Female/Other)
  - Religion (Hindu/Muslim/Christian/Other)
  - City / Country
  - Category
  - Donation Frequency
  - AssignedToUserId
  - Age range (if DOB/age exists)
- Also allow combining filters (AND). (Example: Religion=Muslim AND Gender=Female)

D) “Support Preferences” (NEW donor profile field)
We want broadcasting based on donor interest like groceries/education/medicines etc.
- Add a new multi-select field in Donor Profile:
  - supportPreferences: array/list (multi-select)
  - options: GROCERIES, EDUCATION, MEDICINES, TOILETRIES, SPONSORSHIP, GENERAL
- Show this field in:
  - Add Donor form
  - Edit Donor form
  - Donor Details view
- Add filter support in Broadcasting: filter donors by supportPreferences.
Example: send only to donors who prefer GROCERIES.

E) Donation “Kind Category” (Donation record improvement)
To track what was actually donated:
- Add/ensure donation fields:
  - donationType: CASH or KIND
  - if donationType=KIND, store kindCategory:
    GROCERIES, MEDICINES, TOILETRIES, STATIONERY, CLOTHES, OTHER
  - optional: kindDescription text
- Broadcasting page should optionally support filter:
  - donors who donated kindCategory=X in last N days (if feasible with existing schema; otherwise skip this part for now)

3) Send + Safety
- Provide a “Preview Count” button to show how many donors match before sending.
- Provide “Send Broadcast” button:
  - Send to matched donors in background (queue/job) to avoid blocking UI.
  - Log each send in CommunicationMessage/CommunicationLog table (already exists).
  - Show progress summary: total, sent, failed, skipped (missing phone/email).
- Must validate WhatsApp numbers are E.164 format with + (e.g., +919700700700). If donor has phone without +, normalize it.

4) Admin Permissions
- Only ADMIN (and optionally MANAGER) can access Broadcasting page and send.
- Use existing auth/guards and permission system.

Deliverables:
- New page /dashboard/broadcasting with filters + template selection + send.
- New donor field supportPreferences (multi-select) end-to-end.
- Optional donation kindCategory support (only if easy with current schema).
