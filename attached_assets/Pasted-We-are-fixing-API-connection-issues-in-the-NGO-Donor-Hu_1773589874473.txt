We are fixing API connection issues in the NGO Donor Hub system.

Stack

Frontend
Next.js (apps/web)
Hosted on Vercel

Backend
NestJS (apps/api)
Hosted on Railway

Database
Supabase Postgres
Prisma ORM

---

PROBLEM

Buttons like:

Add Donation
Add Family Member
Add Pledge
Add Special Day

open correctly but data is not saved.

Browser console shows:

ERR_NAME_NOT_RESOLVED

This means the frontend API base URL is incorrect.

---

PART 1 — CREATE CENTRAL API CLIENT

Create file:

apps/web/src/lib/api.ts

```ts
const API_BASE_URL =
 process.env.NEXT_PUBLIC_API_URL ||
 "https://dms-production-598e.up.railway.app"

export async function apiFetch(path: string, options?: RequestInit) {
 const res = await fetch(`${API_BASE_URL}${path}`, {
  headers: { "Content-Type": "application/json" },
  ...options,
 })

 if (!res.ok) throw new Error("API error")

 return res.json()
}
```

---

PART 2 — UPDATE ALL API CALLS

Replace all fetch calls.

Example before:

```
fetch("/api/donations")
```

Replace with:

```
apiFetch("/donations")
```

Example POST:

```
apiFetch("/donations",{
 method:"POST",
 body: JSON.stringify(data)
})
```
