We want to redesign donor-related actions into one unified Donor Action Engine.

FINAL DECISION:
- Merge Birthday Wishes, Reminders, and Follow-ups into Daily Actions -> Task Inbox
- Task Inbox becomes the main place for all donor-related action tasks
- Donor Updates stays separate for content/news/update publishing
- We will support both automatic and manual donor actions

GOAL:
When donor data is entered or donor behavior is detected, the system should automatically generate actionable tasks in Task Inbox.

WE NEED THESE ACTION TYPES:

1. Birthday
- donor birthday task
- auto WhatsApp if donor is eligible
- also create manual task for call/message

2. Anniversary
- donor marriage anniversary task
- auto WhatsApp if eligible
- also create manual task

3. Remembrance
- death anniversary of donor’s loved one
- donor may have multiple reminders
- create sensitive remembrance/follow-up task

4. Pledge Follow-up
- when pledged donation due date comes and donation not received
- create follow-up task
- support both automatic and manual follow-up

5. Smart Donation Reminder
- for monthly / quarterly donors based on donation pattern
- example: if donor usually donates by 6th and current cycle has no donation yet, create reminder task
- this should be driven by donor history / AI insight logic

6. Sponsorship Update
- if donor sponsors a beneficiary, create at least one quarterly task to send update
- task should be linked to donor + beneficiary relationship
- retrieve this from donor sponsorship data automatically

7. Custom Reminder
- manual donor reminder dates if needed

WHATSAPP / MANUAL LOGIC:
- not every donor can receive automated WhatsApp unless they have initiated first
- therefore every donor action should support:
  a) auto_possible = true/false
  b) manual_required = true/false
- if auto is not possible, task should still appear for staff/founder to call or send manual WhatsApp

TIME WINDOWS REQUIRED:
Task Inbox must support and clearly show:
- Today
- Next 7 Days
- Next 15 Days
- Next 30 Days
- Overdue

This is important because donor communication often needs advance preparation.

DONOR OWNER / STAFF OWNER:
Each donor action task must show whose donor it is / who owns the relationship.
Example: if donor belongs to Padma, task row should show that clearly.
This is required so staff know who should contact that donor.

COMMUNICATION HISTORY / LOG:
We also need a donor communication log.
For each donor action task, track:
- how many times donor was reminded/contacted
- last contact date
- contact method (call / WhatsApp auto / WhatsApp manual / SMS / email)
- who contacted
- notes / outcome

This should help us understand whether donor was already contacted and what happened.
