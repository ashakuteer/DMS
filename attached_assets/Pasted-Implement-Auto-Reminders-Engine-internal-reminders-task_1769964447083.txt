Implement Auto Reminders Engine (internal reminders + tasks dashboard). No auto messaging yet.

Scope:
- Generate reminders for Special Days (month/day only) and Follow-up reminders.
- Show them in a "Reminders" dashboard for Admin + Staff.
- No auto email/whatsapp sending in this phase.

1) Data Model
Create (or confirm) a table/model: ReminderTask
Fields:
- id
- donorId (required)
- type enum: BIRTHDAY, ANNIVERSARY, MEMORIAL, FOLLOW_UP
- title (string)
- dueDate (date)  // actual due date for current year
- status enum: OPEN, DONE, SNOOZED
- snoozedUntil (optional date)
- createdByUserId (nullable for system)
- createdAt, updatedAt

2) Reminder Generation Logic
A) Special Days (month/day only):
- For each donor special day (month/day), generate reminders at:
  - 30 days before
  - 7 days before
  - 1 day before
- Create tasks for current year and next upcoming occurrence.

B) Follow-up reminders:
- Use existing “Remind Later” tasks (already created).
- Ensure they appear in reminders list when due.

3) Scheduler
- Create a daily cron job (or background job) that runs once per day:
  - Generates upcoming special-day reminders
  - Avoid duplicates (unique key: donorId + type + dueDate + offset)
  - Does not regenerate tasks already done

4) UI
Create a new menu item: Dashboard → Reminders
Views:
- Today
- Next 7 days
- Next 30 days
- Overdue
Each row shows:
- Donor name
- Task type + title
- Due date
- Suggested action buttons:
  - Copy WhatsApp template
  - Open donor profile
  - Send Email button (manual)
  - Mark Done
  - Snooze 7/30 days

5) Access Control
- Admin + Staff can view and mark done/snooze.
- Telecaller read-only.

6) Logging
- Log when a reminder is generated (system)
- Log when marked done or snoozed.

7) Verification
- Add a donor birthday (month/day)
- Confirm reminders appear for 30/7/1 days logic (simulate dates if needed)
- Confirm no duplicates created on repeated scheduler runs
