Implement automatic donation receipts (PDF + Email) for the existing NGO Donor Management System.

Context:
- Donors and Donations are already working.
- Each donation already has a unique receipt number (AKF-YYYY-YY-XXXXX).
- Use existing PostgreSQL + Prisma setup.
- Do NOT change donor or donation creation logic.

Requirements:

1) PDF Receipt Generation
- Create a backend service to generate a professional PDF receipt for a donation.
- Receipt design should be formal NGO style:
  - Header: “Asha Kuteer Foundation” (logo placeholder at top)
  - Title: “Donation Receipt”
  - Receipt Number
  - Donation Date
  - Donor Name
  - Donation Amount (₹)
  - Amount in words
  - Payment Mode
  - Purpose / Category
  - 80G text:
    “This donation is eligible for deduction under Section 80G of the Income Tax Act, 1961.”
  - Footer:
    “Thank you for supporting Asha Kuteer Foundation”
    “Authorized Signatory”
    “This is a system-generated receipt.”

- PDF should be A4, print-ready.
- PDF should be generated on demand (do not store permanently for now).

2) Email Sending (Automatic)
- After a donation is successfully recorded:
  - Automatically send an email to the donor.
- Email details:
  - From Name: “Asha Kuteer Foundation”
  - Subject: “Donation Receipt – Asha Kuteer Foundation”
  - Email body (simple, warm tone):
    “Thank you for your generous donation. Please find your receipt attached.”
  - Attach the generated PDF receipt.

3) Access & Safety
- Do NOT expose private donor data unnecessarily.
- If donor email is missing, skip email sending gracefully (no crash).

4) Frontend
- No UI changes required now.
- Donation save flow must remain fast (email sending should not block UI).

5) Verification
- Test by recording a new donation.
- Confirm:
  - Donation saves successfully
  - Receipt number is generated
  - Email is sent with PDF attached
  - No existing features are broken

Important:
- Do NOT change database schema unless absolutely required.
- Keep the implementation simple and reliable.
