Feature Request: Soft Delete System for Donors and Beneficiaries

I want to add a **safe soft-delete system** for both:

* Donors
* Beneficiaries

This must follow **production SaaS standards**.

Important Requirements

1. SOFT DELETE (not hard delete)

Records must NOT be permanently removed from database.

Add fields in Prisma models:

```
isDeleted      Boolean   @default(false)
deletedAt      DateTime?
deletedBy      String?
```

Soft delete means:

* isDeleted = true
* deletedAt = current timestamp
* deletedBy = admin user id

All normal queries must automatically filter:

```
where: { isDeleted: false }
```

---

2. ROLE BASED PERMISSION

Only **ADMIN** can delete.

Staff users can only:

* deactivate donors
* block donors
* mark beneficiaries inactive

But **cannot delete records**.

Backend must check role before allowing deletion.

---

3. BACKEND API

Create endpoints:

```
DELETE /donors/:id
DELETE /beneficiaries/:id
```

These endpoints should:

1. Check user role (must be ADMIN)
2. Perform soft delete
3. Return success message

Example service logic:

```
await prisma.donor.update({
 where: { id },
 data: {
   isDeleted: true,
   deletedAt: new Date(),
   deletedBy: adminId
 }
})
```

Do NOT allow deletion if:

* donor has active sponsorship
* donor has important financial records (optional safety)

---

4. DATABASE SAFETY

Update Prisma models:

Donor
Beneficiary

Add:

```
isDeleted Boolean @default(false)
deletedAt DateTime?
deletedBy String?
```

Then ensure all queries filter:

```
isDeleted: false
```

Buttons should be:

Red button:

```

```

before allowing confirmation.
