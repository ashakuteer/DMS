PART 3 — PEOPLE & OCCASIONS SYNC FROM MEALS
==================================================

When saving a Meals Sponsorship that has an occasion, also reflect it in donor People & Occasions.

ONLY do this when:
- occasionType != NONE

DATA TO SYNC
Use these fields from Meals:
- donorId
- occasionType
- occasionFor
- occasionPersonName
- occasionRelationship
- mealServiceDate

TARGET
Create or update an entry in donor People & Occasions / special occasions area so it becomes visible there too.

SYNC RULES
A. If Occasion For = Self
Create/update a People & Occasions entry representing:
- relation = Self
- occasion = mapped from meal occasion
- date = mealServiceDate day/month

B. If Occasion For = Other Person
Create/update using:
- person name = occasionPersonName
- relation = occasionRelationship
- occasion = mapped from meal occasion
- date = mealServiceDate day/month

OCCASION MAPPING
Map meal occasion values safely to People & Occasions equivalents:
- BIRTHDAY → Birthday
- WEDDING_ANNIVERSARY → Wedding Anniversary
- MEMORIAL → Memorial Day / Memorial depending on existing naming
- OTHER → Other

DATE HANDLING
People & Occasions should store the relevant occasion date/month based on mealServiceDate.
If the existing module stores only day/month, convert accordingly.
If it stores a full date, use mealServiceDate.

DUPLICATE PREVENTION (VERY IMPORTANT)
Do NOT create duplicate People & Occasions entries every time a similar meals donation is added.

Use safe matching / upsert logic:
- same donor
- same person name (or self)
- same relation
- same occasion type
- same day/month or same occasion date

If matching record already exists:
- do not duplicate
- either keep existing or update it safely if needed

IMPORTANT
Do NOT overwrite unrelated manually-entered People & Occasions records incorrectly.
Prefer:
- create if not exists
- otherwise leave existing as-is unless minor safe update is needed
