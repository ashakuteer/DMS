=====================================================
A) PLEDGES / PROMISES (NEW – MUST IMPLEMENT)
=====================================================

Use case:
A donor promises future support (money/kind/rice/groceries/visit). We must remember it, track status, remind, and mark fulfilled/modified.

1) Data Model (Prisma)
Create Pledge model linked to donor:
- id
- donorId (required)
- pledgeType enum: MONEY, RICE, GROCERIES, MEDICINES, MEAL_SPONSOR, VISIT, OTHER
- quantity (optional string, e.g. "25kg", "1 bag", "1 month rice")
- amount (optional number for MONEY)
- promisedDate (optional date) OR promisedMonthDay (optional month/day)  // if only month known allow storing month + day = 1
- expectedFulfillmentDate (date, required)  // the “next month” date
- notes (text)
- status enum: PENDING, FULFILLED, POSTPONED, CANCELLED
- statusColor mapping:
  PENDING=yellow, FULFILLED=green, POSTPONED=orange, CANCELLED=red
- fulfilledDonationId (optional) // link if fulfilled by a donation entry
- createdByUserId
- createdAt, updatedAt

2) Backend APIs
- GET /api/pledges?donorId=
- POST /api/pledges
- PATCH /api/pledges/:id (update fields/status)
- POST /api/pledges/:id/mark-fulfilled (optionally link donationId)
- POST /api/pledges/:id/postpone (new expected date + note)
- DELETE /api/pledges/:id (Admin only)

RBAC:
- Admin: full
- Staff: create/update/mark fulfilled/postpone
- Telecaller: view only

Audit log all pledge create/update/status changes.

3) Frontend UI
- Donor Profile has a “Pledges” tab (activate fully).
- Show pledge list with badges (status color).
- Add button “Add Pledge”
- Form fields:
  pledgeType, amount/quantity, expectedFulfillmentDate, notes
- Actions per pledge:
  Mark Fulfilled, Postpone, Cancel
- If donation is recorded for same donor near expected date, show suggestion:
  “Does this fulfill a pledge?” allow linking.

4) Pledge Reminders
- Create reminders for pledges:
  - 7 days before expected date
  - 2 days before
  - On due date
- These appear in Reminders + Daily Actions.
- Auto email: ONLY 2 days before and on due date (optional) OR keep as reminder-only (safe default: reminder-only unless admin enables).
