Decisions you should follow (so it stays simple)
Auth model

Frontend signs in with Google → gets Firebase ID token

Backend (Firebase Functions) verifies token using Admin SDK

Firestore security: you can do either:

Functions-only access (simple, secure, centralized)

Or direct client reads/writes with Firestore Rules (more complex)

I recommend Functions-only first.

Mapping your existing data to Firestore (recommended schema)

Keep it simple like:

users/{uid} (role, name, email)

donors/{donorId}

donations/{donationId}

pledges/{pledgeId}

communications/{commId}

beneficiaries/{beneficiaryId}

campaigns/{campaignId}

Common patterns:

Use createdAt, updatedAt as Firestore timestamps

Create “indexes” only if queries require them (Firestore will tell you)

Storage

Firebase Storage paths like:

donors/{donorId}/photos/{fileName}

beneficiaries/{id}/photos/{fileName}

ngo-documents/{docId}/{version}/file.pdf
